# ADR-0001: Flujo SDD obligatorio

## Estado

accepted

## Contexto

El proyecto frontend ya tiene documentación amplia, pero no tenía una puerta de calidad uniforme para cambios funcionales.

## Decisión

Adoptar flujo Spec-Driven Development obligatorio para cambios de producto:

1. Spec aprobada.
2. Casos de prueba definidos.
3. Trazabilidad actualizada.

## Consecuencias

- Positivas: mejor previsibilidad y menos regresiones.
- Negativas: mayor coste inicial de documentación.
- Riesgos: bypass del flujo en cambios urgentes.
