# Matriz de compatibilidad Frontend/Backend

| Frontend Version | Backend Version | Estado | Notas |
|---|---|---|---|
| 0.1.x | 1.0.x | Compatible | Base MERN separada |

## Reglas de actualización

1. Añadir una fila por release relevante.
2. Marcar `No compatible` cuando cambie contrato sin fallback.
3. Referenciar PRs o ADRs que justifican el cambio.
